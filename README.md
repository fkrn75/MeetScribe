# MeetScribe

> 회의 녹음 파일을 넣으면 **화자를 자동으로 분리**해 읽기 좋은 회의록으로 만들어 주는 **완전 로컬** 데스크탑 앱.

오디오/영상 파일을 드래그&드롭하면 음성을 텍스트로 옮기고(STT), 누가 말했는지 화자별로 나눈 뒤(speaker diarization), 화자별 색상 타임라인과 편집 가능한 회의록으로 보여 줍니다. **모든 처리는 내 PC에서만** 이뤄지며 오디오나 전사 내용은 외부로 전송되지 않습니다.

> ⚠️ **현재 상태: 설계 단계 (제작 전)** — 이 저장소에는 아직 설계·명세 문서만 있습니다. 구현은 [체크리스트](체크리스트.md)의 마일스톤(M0~M7) 순서로 진행할 예정입니다.

## ✨ 주요 특징

- **완전 로컬 / 오프라인** — 첫 모델 다운로드 후 인터넷 없이 동작, 데이터 외부 전송 0
- **한국어 우선** — 한국어 회의 전사·단어 정렬에 최적화
- **자동 화자 분리** — 누가 언제 말했는지 자동 구분, 이름·병합·색상 편집 가능
- **GPU 가속** — NVIDIA GPU에서 빠르게 처리 (CPU 폴백 지원)
- **다양한 내보내기** — txt · srt · docx · json

## 🧱 기술 스택

| 영역 | 선택 |
|------|------|
| 셸 / UI | Tauri 2 (웹 프론트엔드 + Rust) |
| 처리 엔진 | Python 사이드카 — FastAPI 로컬 API + SSE 진행률 |
| STT | WhisperX (faster-whisper `large-v3`) |
| 단어 정렬 | 한국어 wav2vec2 (`kresnik/wav2vec2-large-xlsr-korean`) |
| 화자 분리 | pyannote.audio 3.1 |
| 패키징 | PyInstaller 사이드카 → Tauri 번들(`.msi`/`.exe`) |

처리 흐름: `오디오/영상 → 16kHz WAV 변환 → STT → 단어 정렬 → 화자 분리 → 병합 → 화자별 회의록`

## 🎯 성능 목표

- **GPU(VRAM 8GB급)**: 1시간 회의 ≈ 5~10분 (목표치, 구현 후 검증 예정)
- **장시간(2~3h) 회의**: 청크 단위 화자분리 + 단계 프로세스 격리 + 체크포인트로 완주 보장 (설계서 §10)

## 🗺️ 로드맵 (요약)

- **M0** 환경 셋업 → **M1** 백엔드 코어(CLI 검증) → **M2** API+사이드카 → **M3** 최소 GUI
- **M4** 결과 뷰·화자 편집 → **M5** 내보내기·프로젝트 저장 → **M6** 패키징·배포
- **v2**: LLM 회의 요약·액션아이템, 화자 등록·자동 식별, 실시간 라이브 전사
