1MeetScribe란?

회의 녹음 파일(오디오·영상)을 넣으면 음성을 텍스트로 옮기고(STT), 누가 언제 말했는지 화자별로 나눈 뒤(화자 분리), 화자별 색상 타임라인과 편집 가능한 회의록으로 보여 주는 완전 로컬 데스크탑 앱입니다.

처리 흐름은 다음과 같습니다:

오디오/영상 → 16kHz WAV 변환 → 음성 인식(STT) → 단어 정렬 → 화자 분리 → 병합 → 화자별 회의록
ℹ️ 핵심

오디오와 전사 내용은 외부로 전송되지 않습니다. 인터넷은 처음 모델을 받을 때 한 번만 필요하고, 그 뒤로는 오프라인으로 동작합니다.

3허깅페이스 모델 설정 (필수)

⭐ "다른 사람이 쓰려면 모델을 다시 받아야 하나요?" — 네, 그렇습니다.

AI 모델은 앱에 포함되어 있지 않습니다. 용량이 크고(합쳐서 수 GB), 특히 화자 분리 모델(pyannote)은 gated 모델(약관 동의가 필요)이라 재배포가 불가능합니다. 그래서 각자의 PC에서 처음 실행할 때 자동으로 내려받아 로컬에 캐시합니다(이후 오프라인 동작). 즉 새 PC에서 쓰려면 그 PC에서 모델을 새로 받아야 합니다.

그리고 화자 분리(pyannote)는 각 사용자가 본인 HF 계정으로 약관에 동의하고 본인 토큰을 발급·등록해야 합니다(아래 3-2 절차). 보안상 남의 토큰을 공유해 쓰지 마세요. 정리하면 다른 사람이 쓸 때 필요한 1회 설정은 ① 모델 다운로드(자동) · ② 본인 토큰 발급 · ③ 약관 동의 후 토큰 등록 입니다.

사용하는 모델

모델용도받는 방식
faster-whisper large-v3 음성 인식(STT) 공개 · 첫 실행 시 자동(~3GB)
kresnik/wav2vec2-large-xlsr-korean 한국어 단어 정렬 공개 · 첫 실행 시 자동
pyannote/speaker-diarization-3.1
pyannote/segmentation-3.0		 화자 분리 gated · HF 토큰 + 약관 동의 필요

화자 분리를 쓰려면 (한 번만 설정)

  1. huggingface.co 에 가입합니다(무료).
  2. 아래 두 모델 페이지에 들어가 약관에 동의(Agree / Access repository)합니다:
    • pyannote/speaker-diarization-3.1
    • pyannote/segmentation-3.0
  3. 액세스 토큰을 만듭니다 — 설정 → Access Tokens → New token.
    반드시 classic Read 토큰

    Fine-grained 토큰은 이 모델에서 동작하지 않습니다. 토큰 종류를 "Read"(classic)로 선택하세요.

  4. 토큰을 HF_TOKEN 환경변수로 설정하고 앱을 다시 시작합니다. Windows에서: 
    setx HF_TOKEN "hf_여기에_본인_토큰"
# 설정 후 터미널/앱을 새로 켜야 적용됩니다.
ℹ️ 토큰이 없으면?

음성 인식(STT)과 단어 정렬은 토큰 없이도 동작하지만, 화자 분리는 비활성됩니다(앱 상단에 안내 배너가 뜹니다). 토큰은 코드나 채팅에 붙여넣지 말고 환경변수로만 두세요.

모델 캐시 위치

  • STT · 단어 정렬: ~/.meetscribe/models (환경변수 MEETSCRIBE_CACHE로 변경 가능)
  • 화자 분리(pyannote): 허깅페이스 기본 캐시 ~/.cache/huggingface

처음 한 번만 다운로드하며, 이후에는 인터넷 없이 동작합니다.

💻 다른 컴퓨터에서 쓸 때도 받아야 하나요? — 네

캐시는 계정이 아니라 컴퓨터(로컬 폴더)에 저장됩니다. 같은 사람이라도 새 PC에서는 모델을 다시 받습니다. 단, 약관 동의·토큰은 본인 HF 계정에 묶여 있어 그대로 유효하므로 새 PC엔 HF_TOKEN만 설정하면 됩니다(재동의 불필요).

다시 받기 싫으면 캐시 두 폴더(~/.meetscribe/models · ~/.cache/huggingface)를 새 PC의 같은 위치로 복사하거나, MEETSCRIBE_CACHE를 USB·공유 드라이브 같은 공용 위치로 지정하면 재다운로드를 건너뜁니다.

4설치 & 실행

지금(개발 실행)

  1. 저장소를 클론합니다.
  2. 사이드카(Python) 환경을 만듭니다 — Python 3.12 가상환경에 사이드카 패키지와 의존성 (GPU용 PyTorch, WhisperX, pyannote.audio, faster-whisper 등)을 설치합니다.
  3. 위 3장대로 HF_TOKEN을 설정합니다.
  4. 프론트엔드 의존성: 
    npm install
  5. 개발용 사이드카 바이너리 준비: 
    powershell scripts/setup-dev-sidecar.ps1
    (venv의 사이드카 실행파일을 src-tauri/binaries/로 복사)
  6. 실행: 
    npm run tauri dev
    창이 뜨고, 사이드카(로컬 API)가 자동으로 기동합니다.

환경변수 요약

변수용도필수
HF_TOKEN화자 분리(pyannote) 모델 접근화자 분리에 필수
MEETSCRIBE_CACHE모델 캐시 폴더 변경선택
MEETSCRIBE_DIARIZE_SUBPROCESS1이면 화자분리를 별도 프로세스로 — 2~3시간 장시간 회의의 메모리 부족(OOM) 방지선택
💡 향후 배포 버전

PyInstaller로 사이드카를 동결하고 Tauri로 묶으면 .msi 설치 파일이 됩니다. 그때는 설치 후 HF_TOKEN만 설정하면 첫 실행 시 모델이 자동으로 받아집니다(Python·Node 설치 불필요).

5기본 사용법

  1. 앱을 실행하면 마이크 화면이 나옵니다.
  2. 회의 파일을 창에 드래그&드롭하거나, 마이크를 클릭해 파일을 고릅니다.
    지원 형식: mp3 · m4a · wav · flac · ogg · mp4 · mov · mkv · webm
  3. 변환이 시작되면 진행바와 별 파동 애니메이션이 나타납니다(전처리 → 인식 → 정렬 → 화자 분리 → 병합). 중간에 취소 버튼으로 멈출 수 있습니다(약 10초 안에 중단).
  4. 완료되면 화자별 색상 타임라인 회의록과 화자 패널이 표시됩니다.
  5. 화자 편집: 화자 패널에서 이름 바꾸기 · 병합 · 색상 변경.
  6. 내보내기: 원하는 형식(TXT · SRT · DOCX · JSON)을 누르고 저장 위치를 고릅니다.
💡 처리 속도

VRAM 8GB급 GPU 기준 1시간 회의 ≈ 5~10분. CPU만 쓰면 훨씬 오래 걸립니다. 첫 실행은 모델 다운로드 때문에 추가 시간이 듭니다(한 번만).
