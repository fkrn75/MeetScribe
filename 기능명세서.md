# MeetScribe 세부 기능명세서 (Functional Spec)

> **작성일**: 2026-06-17 · **상태**: 설계 단계
> 우선순위: **P0**=MVP 필수 · **P1**=MVP 권장 · **P2**=v2 이후
> 각 기능은 *입력 / 처리 / 출력 / UI / 수용기준(AC)* 으로 명세

## F1. 파일 입력 `P0`
- **입력**: 오디오/비디오 파일 (mp3, m4a, wav, flac, mp4, mov 등)
- **처리**: 형식·길이·손상 검증, 메타데이터(길이·샘플레이트) 추출
- **출력**: 유효 파일 경로 + 메타
- **UI**: 드래그&드롭 존, "파일 선택" 버튼, 최근 항목
- **AC**:
  - [ ] 지원 형식 드롭 시 메타 표시
  - [ ] 미지원/손상 파일은 명확한 에러 메시지
  - [ ] 2GB 이상 대용량도 경로 전달(메모리 복사 없음)

## F2. 오디오 전처리 `P0`
- **입력**: 원본 파일 경로
- **처리**: ffmpeg로 **16kHz mono WAV** 변환, (옵션) 노이즈/무음 트리밍
- **출력**: 표준화된 임시 WAV
- **AC**:
  - [ ] 비디오에서 오디오 트랙만 추출
  - [ ] 변환 실패 시 원인 로깅
  - [ ] 임시 파일은 작업 종료 후 정리

## F3. STT (음성→텍스트) `P0`
- **입력**: 표준 WAV, 언어(기본 `ko`), 모델(기본 large-v3)
- **처리**: faster-whisper 전사, VAD 필터, `condition_on_previous_text=False`(환각 억제)
- **출력**: 세그먼트별 텍스트 + 대략 타임스탬프
- **UI**: 설정에서 모델·언어 선택
- **AC**:
  - [ ] 한국어 회의 전사 정상 동작
  - [ ] 언어 자동감지 옵션 제공
  - [ ] GPU/CPU에 따라 모델·compute_type 자동 조정

## F4. 화자 분리 `P0`
- **입력**: 표준 WAV, (옵션) 최소/최대 화자 수 힌트
- **처리**: pyannote.audio 3.1 파이프라인 → 화자별 발화 구간(turn)
- **출력**: `[{speaker, start, end}]`
- **UI**: 설정에서 화자 수 힌트(자동/수동) 입력
- **AC**:
  - [ ] 화자 수 미지정 시 자동 추정
  - [ ] min/max 지정 시 반영
  - [ ] HF 토큰 없으면 안내 후 graceful 실패

## F5. 정렬·병합 `P0`
- **입력**: STT 단어, 화자 구간
- **처리**: wav2vec2(ko) 단어 정렬 → 단어별 정확 타임스탬프 → 화자 구간과 매칭
- **출력**: 최종 세그먼트 `[{speaker, start, end, text, words[]}]`
- **AC**:
  - [ ] 각 발화에 화자가 부여됨
  - [ ] 겹침 구간 합리적 처리(주 화자 할당)
  - [ ] 단어 단위 타임스탬프 보존

## F6. 진행률 / 취소 `P0`
- **처리**: 단계별 가중치 진행률을 SSE로 푸시, 취소 시 작업 중단·정리
- **출력**: `{stage, percent, message, eta}`
- **UI**: 진행바, 현재 단계 텍스트, 경과/예상 시간, **취소** 버튼
- **AC**:
  - [ ] 단계 전환이 UI에 반영
  - [ ] 취소 시 즉시 중단 + 임시파일 정리
  - [ ] 긴 작업에도 UI 멈춤 없음(비차단)

## F7. 결과 뷰 (타임라인) `P0`
- **입력**: 세그먼트 JSON
- **UI**: 화자별 색상 구분 트랜스크립트 (시각·화자칩·텍스트 행), 스크롤·검색
- **AC**:
  - [ ] 화자별 색상 일관 표시
  - [ ] 타임스탬프 클릭 → 해당 위치
  - [ ] 텍스트 내 검색·하이라이트

## F8. 화자 편집 (이름·병합·색상) `P0`
- **처리**: SPEAKER_00 → "김부장" 매핑, 동일인 화자 **병합**, 색상 변경
- **UI**: 화자 패널 목록, 인라인 이름 편집, 병합 드래그/버튼
- **AC**:
  - [ ] 이름 변경이 전체 트랜스크립트에 즉시 반영
  - [ ] 두 화자 병합 → 모든 발화 통합
  - [ ] 변경 undo 가능

## F9. 텍스트 편집 `P1`
- **처리**: 오타·오인식 수정, 세그먼트 분할/병합
- **UI**: 텍스트 행 클릭 → 인라인 편집
- **AC**:
  - [ ] 편집 내용 저장·반영
  - [ ] undo/redo 지원

## F10. 내보내기 `P0`
- **입력**: (편집된) 세그먼트, 형식
- **형식**:
  | 형식 | 내용 |
  |------|------|
  | **txt** | `[00:12] 김부장: ...` 평문 회의록 |
  | **srt** | 자막(화자 prefix 포함) |
  | **docx** | 서식 있는 회의록(화자별 문단·색상) |
  | **json** | 원본 데이터(재가공용) |
- **AC**:
  - [ ] 각 형식 정상 생성·열림
  - [ ] 화자 이름·타임스탬프 반영
  - [ ] 저장 위치 선택

## F11. 프로젝트 저장/불러오기 `P1`
- **처리**: `.msz`(ZIP: project.json + 오디오 옵션) 저장·로드
- **AC**:
  - [ ] 저장 후 재로드 시 편집·화자 상태 복원
  - [ ] 오디오 포함/제외 옵션

## F12. 설정 `P0`
- **항목**: 모델 선택, GPU/CPU 모드, **HF 토큰**, 모델 캐시 경로, 기본 언어, 화자 수 기본값
- **AC**:
  - [ ] HF 토큰 입력·검증
  - [ ] 설정 영속화(재시작 유지)
  - [ ] 모델 다운로드 상태·용량 표시

## F13. 오디오 재생 연동 `P1`
- **처리**: 세그먼트 클릭 → 해당 구간 재생, 재생 위치 ↔ 트랜스크립트 동기화
- **AC**:
  - [ ] 발화 클릭 시 정확한 위치 재생
  - [ ] 재생 중 현재 발화 하이라이트

## F14. LLM 회의 요약·액션아이템 `P2`
- **처리**: 전사문 → LLM(Claude API 등) → 요약·결정사항·액션아이템·화자 이름 추론
- **AC**:
  - [ ] 요약·액션아이템 생성
  - [ ] API 키 설정(로컬 LLM 옵션 고려)
  - [ ] 결과를 내보내기에 포함

## F15. 화자 등록·자동 식별 `P2`
- **처리**: 사전 등록된 목소리 임베딩과 대조 → 자동 이름 부여
- **AC**:
  - [ ] 화자 목소리 등록
  - [ ] 신규 회의에서 자동 매칭

## F16. 실시간 라이브 전사 `P2`
- **처리**: 마이크/시스템 오디오 스트리밍 → diart 등 실시간 화자분리
- **AC**:
  - [ ] 라이브 캡처
  - [ ] 준실시간 화자별 표시

## 비기능 요구 (NFR)

| ID | 항목 | 기준 |
|----|------|------|
| NFR1 | 오프라인 | 첫 모델 다운로드 후 인터넷 없이 동작 |
| NFR2 | 프라이버시 | 오디오·전사 데이터 외부 전송 0 |
| NFR3 | 속도(GPU) | 본인 VRAM 8GB급 기준 1시간 회의 약 5~10분(STT+화자분리 합산) |
| NFR4 | 속도(CPU) | 동작 보장(medium), 진행률·예상시간 제공 |
| NFR5 | 안정성 | **1h: 무크래시·취소가능 / 2~3h: 청크 diarization+프로세스격리+체크포인트로 완주 보장** (통짜 1패스는 pyannote 런타임 메모리 OOM으로 8GB서 붕괴 — 설계서 §10 적대검증) |
| NFR7 | 장시간 처방 | 청크 단위 화자분리, 단계 프로세스 격리, 단계별 체크포인트, 정렬 실패 graceful degrade (MVP 필수) |
| NFR6 | 설치 | 모델 비동봉으로 인스톨러 경량 |
